Tools and Toolkits form the core functionality of Composio, enabling your applications to interact with external services. Understanding the distinction between these concepts and how to use them effectively is essential for building robust integrations.

## Understanding the Hierarchy

### Tools: Individual Actions

A **Tool** represents a single, specific action that can be performed with an external service. Each tool has:

- **Unique Slug**: A standardized identifier (e.g., `GITHUB_GET_REPO`)
- **Input Parameters**: The data required to execute the tool
- **Output Schema**: The structure of data returned
- **Authentication Requirements**: Whether user authorization is needed

<CodeGroup>
```python Python
# Example: Getting repository information
result = composio.tools.execute('GITHUB_GET_REPO',
    user_id='user_123',
    arguments={
        'owner': 'composiohq',
        'repo': 'composio'
    }
)

# The tool returns structured data

print(result.data.name) // "sdk"
print(result.data.description) // Repository description
print(result.data.stargazers_count) // Number of stars

````

```typescript TypeScript
// Example: Getting repository information
const result = await composio.tools.execute('GITHUB_GET_REPO', {
  userId: 'user_123',
  arguments: {
    owner: 'composiohq',
    repo: 'composio'
  }
});

// The tool returns structured data
console.log(result.data.name);          // "sdk"
console.log(result.data.description);   // Repository description
console.log(result.data.stargazers_count); // Number of stars
````

</CodeGroup>

### Toolkits: Service/App Collections

A **Toolkit** is a collection of related tools that work with the same external service. For example:

- **GitHub Toolkit**: Contains tools for repositories, issues, pull requests, users, organizations
- **Gmail Toolkit**: Contains tools for reading, sending, searching emails
- **Slack Toolkit**: Contains tools for messaging, channels, user management

<CodeGroup>
```python Python
# Get all tools from the GitHub toolkit
github_tools = composio.tools.get('user_123',
    toolkits=['github']
)

print(f"GitHub toolkit contains {len(github_tools)} tools")

# Outputs: GitHub toolkit contains 45+ tools

````

```typescript TypeScript
// Get all tools from the GitHub toolkit
const githubTools = await composio.tools.get('user_123', {
  toolkits: ['github']
});

console.log(`GitHub toolkit contains ${githubTools.length} tools`);
// Outputs: GitHub toolkit contains 45+ tools
````

</CodeGroup>

## Tool Anatomy

### Tool Structure

Every tool follows a consistent structure:

<CodeGroup>
```json JSON
{
  "slug": "GITHUB_GET_REPO",
  "name": "Get Repository",
  "description": "Retrieve information about a specific GitHub repository",
  "inputParameters": {
    "type": "object",
    "properties": {
      "owner": {
        "type": "string",
        "description": "Repository owner username or organization name"
      },
      "repo": {
        "type": "string", 
        "description": "Repository name"
      }
    },
    "required": ["owner", "repo"]
  },
  "outputParameters": {
    "type": "object",
    "properties": {
      "name": { "type": "string" },
      "description": { "type": "string" },
      "stargazers_count": { "type": "integer" },
      "html_url": { "type": "string" }
    }
  }
}
```
</CodeGroup>

### Tool Execution Patterns

**Basic Execution:**

<CodeGroup>

```python Python
try:
    result = await composio.tools.execute('GITHUB_GET_REPO',
        user_id='user_123',
        arguments={
            'owner': 'composio',
            'repo': 'sdk'
        }
    )

    if result.successful:
        print('Repository data:', result.data)
    else:
        print('Tool execution failed:', result.error)
except Exception as error:
    print('SDK error:', str(error))
```

```typescript TypeScript
try {
  const result = await composio.tools.execute('GITHUB_GET_REPO', {
    userId: 'user_123',
    arguments: {
      owner: 'composio',
      repo: 'sdk',
    },
  });

  if (result.successful) {
    console.log('Repository data:', result.data);
  } else {
    console.error('Tool execution failed:', result.error);
  }
} catch (error) {
  console.error('SDK error:', error.message);
}
```

</CodeGroup>

## Toolkit Authentication Patterns

### OAuth2 Toolkits

Most modern services use OAuth2 for secure authentication:

<CodeGroup>
```python Python
# Initiate OAuth flow
connection_request = await composio.toolkits.authorize('user_123', 'github')

# User visits connection_request.redirect_url to authorize

# After authorization, get the connected account

connected_account = await composio.connected_accounts.wait_for_connection(
connection_request.id
)

# Use the connected account with toolkit tools

result = await composio.tools.execute('GITHUB_GET_REPO',
user_id='user_123',
connected_account_id=connected_account.id,
arguments={'owner': 'composio', 'repo': 'sdk'}
)

````
```typescript TypeScript
// Initiate OAuth flow
const connectionRequest = await composio.toolkits.authorize('user_123', 'github');

// User visits connectionRequest.redirectUrl to authorize
// After authorization, get the connected account
const connectedAccount = await composio.connectedAccounts.waitForConnection(
connectionRequest.id
);

// Use the connected account with toolkit tools
const result = await composio.tools.execute('GITHUB_GET_REPO', {
userId: 'user_123',
connectedAccountId: connectedAccount.id,
arguments: { owner: 'composio', repo: 'sdk' }
});

````

</CodeGroup>

### Error Handling Patterns

Different types of errors require different handling approaches:


<CodeGroup>
```python Python
async def execute_tool_safely(tool_slug: str, params: dict):
    try:
        result = await composio.tools.execute(tool_slug, **params)

        if result.successful:
            return result.data
        else:
            # Tool-level error (e.g., API rate limit, invalid parameters)
            raise ToolExecutionError(result.error)
    except ComposioToolNotFoundError:
        # Tool doesn't exist
        print('Tool not found:', tool_slug)
        raise
    except ComposioConnectedAccountNotFoundError:
        # User hasn't connected the required account
        print('Account not connected for toolkit')
        raise
    except Exception as error:
        # Other SDK errors
        print('Unexpected error:', str(error))
        raise
```

```typescript TypeScript
async function executeToolSafely(toolSlug: string, params: Record<string, any>) {
  try {
    const result = await composio.tools.execute(toolSlug, params);
    
    if (result.successful) {
      return result.data;
    } else {
      // Tool-level error (e.g., API rate limit, invalid parameters)
      throw new ToolExecutionError(result.error);
    }
  } catch (error) {
    if (error instanceof ComposioToolNotFoundError) {
      // Tool doesn't exist
      console.error('Tool not found:', toolSlug);
    } else if (error instanceof ComposioConnectedAccountNotFoundError) {
      // User hasn't connected the required account
      console.error('Account not connected for toolkit');
    } else {
      // Other SDK errors
      console.error('Unexpected error:', error);
    }
    throw error;
  }
}
```

## Best Practices

### Tool Selection

- **Be Specific**: Request only the tools you need rather than entire toolkits
- **Use Search**: Leverage search functionality to find relevant tools
- **Check Requirements**: Verify authentication requirements before using tools

### Error Handling

- **Check Success**: Always verify `result.successful` before using data
- **Handle Missing Connections**: Gracefully handle cases where users haven't connected accounts
- **Validate Inputs**: Ensure required parameters are provided

### Performance

- **Batch Operations**: Group related tool executions when possible
- **Cache Results**: Store frequently accessed data to reduce API calls
- **Rate Limiting**: Be aware of external service rate limits

### Security

- **User Context**: Always execute tools in the correct user context
- **Parameter Validation**: Validate user inputs before passing to tools
- **Error Information**: Be careful not to expose sensitive error details to end users
