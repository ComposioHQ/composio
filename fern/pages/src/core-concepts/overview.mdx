Understanding Composio's core concepts is essential for building robust applications. This guide explains the fundamental building blocks and how they work together to provide seamless integration with external services.

## Essential Concepts

### 1. **User IDs** - The Foundation

User IDs are the cornerstone of Composio's security model. Every operation requires a User ID to ensure proper data isolation. You can use the default user ID `default` for public APIs, or create a user ID for each user in your application.

<CodeGroup>
```python Python
# ‚úÖ Each user has their own isolated context
user_a_tools = await composio.tools.get('user_a', toolkits=['github'])
user_b_tools = await composio.tools.get('user_b', toolkits=['github'])
```

```typescript TypeScript
// ‚úÖ Each user has their own isolated context
const userATools = await composio.tools.get('user_a', { toolkits: ['github'] });
const userBTools = await composio.tools.get('user_b', { toolkits: ['github'] });
```
</CodeGroup>

**Why User IDs Matter:**

- **Security**: Prevents users from accessing each other's data
- **Isolation**: Each user's connected accounts are separate
- **Scaling**: Supports multi-tenant applications
- **Compliance**: Meets data privacy requirements

[**‚Üí Learn more about User IDs**](./user-ids)


### 2. **Tools** - Individual Actions

Tools are specific actions you can perform, like "Get GitHub Repository" or "Send Email". Each tool has defined inputs and outputs.

<CodeGroup>
```python Python
# A tool execution
result = await composio.tools.execute('GITHUB_GET_REPO', 
    user_id='user_123',
    arguments={
        'owner': 'composio',
        'repo': 'sdk'
    }
)
```

```typescript TypeScript
// A tool execution
const result = await composio.tools.execute('GITHUB_GET_REPO', {
  userId: 'user_123',
  arguments: {
    owner: 'composio',
    repo: 'sdk',
  },
});
```
</CodeGroup>

**Tool Properties:**

- **Slug**: Unique identifier (e.g., `GITHUB_GET_REPO`)
- **Input Parameters**: What data the tool needs
- **Output**: What data the tool returns
- **Authentication**: Whether the tool requires user authorization


### 3. **Toolkits** - Collections of Tools

Toolkits group related tools from the same service. For example, the GitHub toolkit contains tools for repositories, issues, pull requests, and more.

<CodeGroup>
```python Python
# Get all tools from the GitHub toolkit
github_tools = await composio.tools.get('user_123', 
    toolkits=['github']
)

# Available toolkits include: gmail, slack, notion, hubspot, salesforce, etc.
```

```typescript TypeScript
// Get all tools from GitHub toolkit
const githubTools = await composio.tools.get('user_123', {
  toolkits: ['github'],
});

// Available toolkits include: gmail, slack, notion, hubspot, salesforce, etc.
```
</CodeGroup>

**Popular Toolkits:**

- **GitHub**: Repository management, issues, PRs
- **Gmail**: Email sending, reading, searching
- **Slack**: Messaging, channels, user management
- **Google Calendar**: Event management, scheduling
- **Notion**: Database operations, page creation

[**‚Üí Explore Tools & Toolkits**](./tools-and-toolkits)


### 4. **Connected Accounts** - User Authorizations

Connected Accounts represent a user's authorized connection to an external service. Users must authorize access before tools can act on their behalf.

<CodeGroup>
```python Python
# User authorizes GitHub access
connection_request = await composio.connected_accounts.initiate('user_123', 'ac_authconfig_id')

# After user completes OAuth flow
connected_account = await composio.connected_accounts.wait_for_connection(connection_request.id)

# Now tools can access user's GitHub data
repos = await composio.tools.execute('GITHUB_LIST_REPOS',
    user_id='user_123',
    connected_account_id=connected_account.id
)
```

```typescript TypeScript
// User authorizes GitHub access
const connectionRequest = await composio.connectedAccounts.initiate('user_123', 'ac_authconfig_id');

// After user completes OAuth flow
const connectedAccount = await composio.connectedAccounts.waitForConnection(connectionRequest.id);

// Now tools can access user's GitHub data
const repos = await composio.tools.execute('GITHUB_LIST_REPOS', {
  userId: 'user_123',
  connectedAccountId: connectedAccount.id,
});
```
</CodeGroup>

**Connection Types:**

- **OAuth2**: Most common (GitHub, Google, Slack)
- **API Keys**: For services like OpenAI, Anthropic
- **Basic Auth**: Username/password authentication
- **Custom**: Service-specific authentication methods

[**‚Üí Learn about Connected Accounts**](./connected-accounts)


### 5. **Auth Configs** - Authentication Templates

Auth Configs define how authentication works for each toolkit. They specify the authentication method and required credentials.

<CodeGroup>
```python Python
# Create a custom auth config for GitHub
auth_config = await composio.auth_configs.create('github', {
    'type': 'use_composio_managed_auth',
    'name': 'My GitHub Integration'
})
```

```typescript TypeScript
// Create a custom auth config for GitHub
const authConfig = await composio.authConfigs.create('github', {
  type: 'use_composio_managed_auth',
  name: 'My GitHub Integration',
});
```
</CodeGroup>

**Auth Config Types:**

- **Composio Managed**: Composio handles OAuth apps and credentials
- **User Credentials**: Users provide their own API keys/credentials
- **Custom Auth**: Custom authentication flows
- **No Auth**: For public APIs that don't require authentication

[**‚Üí Understanding Auth Configs**](./auth-configs)


### 6. **Providers** - AI Platform Integration

Providers adapt Composio tools for different AI platforms, handling the formatting and execution patterns each platform expects.

<CodeGroup>
```python Python
from composio.providers.openai import OpenAIProvider
from composio.providers.langchain import LangChainProvider

# OpenAI Provider (non-agentic)
openai_composio = Composio(
    provider=OpenAIProvider()
)

# LangChain Provider (agentic)
langchain_composio = Composio(
    provider=LangChainProvider()
)
```

```typescript TypeScript
import { OpenAIProvider } from '@composio/openai';
import { LangChainProvider } from '@composio/langchain';

// OpenAI Provider (non-agentic)
const openaiComposio = new Composio({
  provider: new OpenAIProvider(),
});

// LangChain Provider (agentic)
const langchainComposio = new Composio({
  provider: new LangChainProvider(),
});
```
</CodeGroup>

**Provider Types:**

- **Non-Agentic**: Format tools for external use (OpenAI, Anthropic, Google)
- **Agentic**: Control tool execution internally (LangChain, Vercel AI)

[**‚Üí Learn about Providers**](./providers)

## Key Patterns and Best Practices

### 1. **User-Centric Design**

<CodeGroup>
```typescript TypeScript
// ‚úÖ Always scope operations to specific users
const userTools = await composio.tools.get(userId, { toolkits: ['github'] });

// ‚ùå Don't use 'default' in production
const defaultTools = await composio.tools.get('default', { toolkits: ['github'] });
```

```python Python
# ‚úÖ Always scope operations to specific users
user_tools = await composio.tools.get(user_id, toolkits=['github'])

# ‚ùå Don't use 'default' in production
default_tools = await composio.tools.get('default', toolkits=['github'])
```
</CodeGroup>

### 2. **Error Handling**

<CodeGroup>
```typescript TypeScript
try {
  const result = await composio.tools.execute('GITHUB_GET_REPO', {
    userId: user.id,
    arguments: { owner: 'composio', repo: 'sdk' },
  });

  if (result.successful) {
    // Handle success
  } else {
    // Handle tool-level errors
    console.error('Tool error:', result.error);
  }
} catch (error) {
  // Handle SDK-level errors
  console.error('SDK error:', error.message);
}
```

```python Python
try:
    result = await composio.tools.execute('GITHUB_GET_REPO',
        user_id=user.id,
        arguments={'owner': 'composio', 'repo': 'sdk'}
    )

    if result.successful:
        # Handle success
        pass
    else:
        # Handle tool-level errors
        print('Tool error:', result.error)
except Exception as error:
    # Handle SDK-level errors
    print('SDK error:', str(error))
```
</CodeGroup>

### 3. **Connection Management**

<CodeGroup>
```typescript TypeScript
// Check if user has required connections
const accounts = await composio.connectedAccounts.list({
  userIds: [userId],
  toolkitSlugs: ['github'],
});

if (accounts.items.length === 0) {
  // Prompt user to connect GitHub
  const connection = await composio.toolkits.authorize(userId, 'github');
  // Redirect to connection.redirectUrl
}
```

```python Python
# Check if user has required connections
accounts = await composio.connected_accounts.list(
    user_ids=[user_id],
    toolkit_slugs=['github']
)

if len(accounts.items) == 0:
    # Prompt user to connect GitHub
    connection = await composio.toolkits.authorize(user_id, 'github')

```
</CodeGroup>

## Common Questions

**Q: Can one user have multiple connected accounts for the same service?**
A: Yes, but it requires explicitly allowing multiple connections when creating them.

**Q: Do I need to store authentication tokens?**
A: No, Composio manages all tokens securely. You only work with Connected Account IDs.

**Q: Can I use tools without user authentication?**
A: Some tools (like HackerNews) don't require authentication, but most do.

**Q: How do I know which tools require authentication?**
A: Tools that require authentication will throw an error if no connected account is provided.
