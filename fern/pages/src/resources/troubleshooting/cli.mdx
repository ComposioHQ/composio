Common issues and solutions for the Composio CLI.

## Command not found

If you're getting "command not found" when trying to run `composio`:

### Verify installation

Check if the CLI is installed:

```bash
which composio
```

This should return the path to the composio executable, typically `~/.composio/composio`.

### Check PATH

Ensure your PATH includes the Composio installation directory:

```bash
echo $PATH | grep .composio
```

If not present, add it to your shell configuration:

```bash
# For bash
echo 'export PATH="$HOME/.composio:$PATH"' >> ~/.bashrc
source ~/.bashrc

# For zsh  
echo 'export PATH="$HOME/.composio:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### Reinstall

If the CLI isn't installed or PATH issues persist:

```bash
# Remove existing installation
rm -f ~/.composio/composio

# Reinstall
curl -fsSL https://composio.dev/install | bash
```

Or using wget:
```bash
wget -qO- https://composio.dev/install | bash
```

## Authentication errors

### Invalid API key

If you're getting authentication errors:

1. Check your current authentication status:
```bash
composio whoami
```

2. If no API key is found or it's invalid, re-authenticate:
```bash
composio logout
composio login
```

### CI/CD authentication

For automated environments, set the API key via environment variable:

```bash
export COMPOSIO_API_KEY="your-api-key"
```

Common issues:
- Ensure the API key is properly exported
- Check for typos in the environment variable name
- Verify the API key hasn't expired

## Generate command issues

### Project type not detected

The CLI auto-detects project type by looking for:
- **TypeScript**: `package.json`, `tsconfig.json`
- **Python**: `requirements.txt`, `setup.py`, `pyproject.toml`

If auto-detection fails:

1. Use language-specific commands:
```bash
composio ts generate  # For TypeScript
composio py generate  # For Python
```

2. Ensure you're in the project root directory:
```bash
cd /path/to/your/project
composio generate
```

### Output directory issues

If generated files aren't appearing where expected:

1. Specify output directory explicitly:
```bash
composio generate --output-dir ./my-types
```

2. Check write permissions:
```bash
ls -la ./  # Unix/Linux/macOS
dir       # Windows
```

### Cache issues

If you're getting stale or outdated type definitions:

1. Clear the cache:
```bash
rm -rf ~/.composio/*.json
```

2. Force fresh generation:
```bash
FORCE_USE_CACHE=false composio generate
```

## Network issues

### Behind a proxy

Configure proxy settings:

```bash
export HTTP_PROXY=http://proxy.company.com:8080
export HTTPS_PROXY=http://proxy.company.com:8080
```

### API connection errors

If you can't connect to Composio API:

1. Check base URL configuration:
```bash
echo $COMPOSIO_BASE_URL
```

2. Reset to default if needed:
```bash
unset COMPOSIO_BASE_URL
composio login
```

## Version issues

### Outdated CLI

Check your current version:
```bash
composio version
```

Upgrade to latest:
```bash
composio upgrade
```

### Version mismatch with SDK

Ensure CLI and SDK versions are compatible:

```bash
# Check CLI version
composio version

# Check SDK version (TypeScript)
npm list @composio/core

# Check SDK version (Python)
pip show composio
```

## Logging and debugging

### Enable debug logging

For detailed output during troubleshooting:

```bash
composio --log-level debug [command]
```

Available log levels:
- `all` - Most verbose
- `trace`
- `debug`
- `info`
- `warning`
- `error`
- `fatal`
- `none` - Silent

### View configuration

Check current configuration and environment:

```bash
# View stored user data
cat ~/.composio/user_data.json

# Check all Composio environment variables
env | grep COMPOSIO
```

## Common error messages

### "API key not found"

Run `composio login` to authenticate.

### "Failed to detect project type"

Ensure you're in a valid Python or TypeScript project directory, or use language-specific commands.

### "Permission denied"

Check write permissions for the output directory or use `sudo` if necessary (not recommended for global installs).

### "Network timeout"

Check your internet connection and proxy settings if applicable.
