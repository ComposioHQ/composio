## üöÄ New Feature: Composio Connect Link

We're excited to introduce **Composio Connect Link**, a revolutionary authentication method that simplifies how your users connect their accounts to your applications. This new feature provides a hosted authentication flow that eliminates the complexity of handling OAuth redirects and authentication states in your application.

### What is Composio Connect Link?

Composio Connect Link creates a secure, hosted authentication page where your users can connect their accounts through Composio's managed OAuth flow. Instead of building and maintaining your own OAuth implementation, you can now provide users with a seamless, one-click authentication experience.

### How It Works

#### 1. Create a Connection Link

<CodeGroup>
```python Python
from composio import Composio

composio = Composio(api_key="YOUR_COMPOSIO_API_KEY")

# Create a connection request
connection_request = composio.connected_accounts.link(
    user_id="user_123",
    auth_config_id="ac_UdqFwixfy3cV"
)

# Get the authentication URL
auth_url = connection_request.redirect_url
print(f"Visit: {auth_url}")
```

```typescript TypeScript  
import { Composio } from '@composio/core';

const composio = new Composio({apiKey: "YOUR_COMPOSIO_API_KEY"});

// Create a connection request
const connectionRequest = await composio.connectedAccounts.link(
    'user_123',
    'ac_UdqFwixfy3cV'
);

// Get the authentication URL
const authUrl = connectionRequest.redirectUrl;
console.log(`Visit: ${authUrl}`);
```
</CodeGroup>

#### 2. Wait for Connection

<CodeGroup>
```python Python
# Wait for the user to complete authentication
connected_account = connection_request.wait_for_connection()
print(f"Connection established: {connected_account.id}")
```

```typescript TypeScript
// Wait for the user to complete authentication  
const connectedAccount = await connectionRequest.waitForConnection();
console.log(`Connection established: ${connectedAccount.id}`);
```
</CodeGroup>

#### 3. Custom Callback URLs (Optional)

<CodeGroup>
```python Python
# Redirect users to your application after authentication

```

### Migration Guide

If you're currently using the `initiate()` method, migrating to Connect Link is straightforward:

**Before (Manual OAuth):**
```typescript
const connectionRequest = await composio.connectedAccounts.initiate(
    userId,
    authConfigId
);
```

**After (Connect Link):**
```typescript
const connectionRequest = await composio.connectedAccounts.link(
    userId, 
    authConfigId
);
```

The response object and `waitForConnection()` method remain the same, ensuring minimal code changes.

### API Reference

The new `link()` method is now available in the Connected Accounts API:

- **Parameters:**
  - `userId` (string): Unique identifier for the user
  - `authConfigId` (string): Auth configuration ID from your dashboard  
  - `options` (optional): Configuration options including `callbackUrl`

- **Returns:** `ConnectionRequest` object with:
  - `redirectUrl`: The hosted authentication URL
  - `waitForConnection()`: Method to await connection completion
  - `id`: Connection request identifier

### Getting Started

1. **Create an Auth Config** in your [Composio Dashboard](https://platform.composio.dev/auth-configs)
2. **Use the new `link()` method** in your application
3. **Direct users** to the generated `redirectUrl`
4. **Wait for connection** using `waitForConnection()`
