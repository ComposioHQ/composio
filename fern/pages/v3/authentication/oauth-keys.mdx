When scaling up to many users or in need of more control, you may want to use your own OAuth credentials.

- [ ]  CLI is just placeholder, but it can actually be a step-by-step CLI thing that guides you through the process of creating an auth config with your own OAuth credentials.
<Tabs>
<Tab title="SDK">
<CodeGroup>
```typescript
import { Composio } from "@composio/core";

const composio = new Composio({
  apiKey: process.env.COMPOSIO_API_KEY,
});

const userId = "your@email.com";

const authConfig = await composio.authConfigs.create("github", {
  authScheme: "OAUTH2",
  type: "use_custom_auth",
  credentials: {
    client_id: "1234567890",
    client_secret: "1234567890",
  },
});

console.log(authConfig.id);
```
</CodeGroup>

Now this auth config can be used to authenticate and execute tools for a given user.

<CodeGroup>
```typescript
const connectionRequest = await composio.connectedAccounts.initiate(
    userId,
    authConfig.id
);

console.log(connectionRequest.redirectUrl);
await connectionRequest.waitForConnection();
  
const result = await composio.tools.execute("GITHUB_STAR_A_REPOSITORY_FOR_THE_AUTHENTICATED_USER", {
  userId,
  arguments: {
    owner: "composiohq",
    repo: "composio",
  },
});
```
</CodeGroup>

</Tab>
<Tab title="CLI">

composio auth-config create --name "GitHub" --oauth-app-id "1234567890" --oauth-app-secret "1234567890"
